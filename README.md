先聊幾個有趣的問題

問：為啥我學完了C語言或者C++，卻還是啥東西也做不出來？

答：程式語言學完了就能做出東西那也真是天才哇！應該說語言學得就算再精通，它其實也只代表完成了「最小的」那一部分，和實際上手幹活之間還是有一個非常大的鴻溝，這個鴻溝就表現為下文即將詳述的編程基礎四大件+應用實踐編程。

問：為什麼C/C++寫出來的東西都是運行於黑乎乎的命令行？這玩意真有用嗎？

答：嘿嘿，誰說黑乎乎的命令行里運行的程序就沒有用！咱大名鼎鼎的Linux系統都以命令行跟用戶交互呢，而且Linux裡面很多強大的工具都是運行於黑乎乎的命令行！

問：從技術學習和實際運用的角度來看，C/C++和Java到底區別在哪？

答：C/C++，它和Java確實不太一樣。C語言和C++，尤其C++，語言粒度細、機制多，性能雖然高，但語言本身的包袱也確實重，我們更願意稱它「造輪子」的語言！也正是因為C語言和C++性能好、粒度細，所以什麼都能做。而Java本身就是一種服務於網際網路軟體開發（後端開發+客戶端開發）的語言，它有一個明顯的「生態圈」的概念，所以應用領域非常清晰。我個人覺得C語言和C++編程比Java還是要難一些，Java畢竟是純應用層的，C語言和C++則對程式設計師能力的要求要更高一些。


崗位分析

不管怎樣，下面即將要介紹的這些學習路線和內容適用於以上所有情況。所以呢，下面就來講講具體的學習路線。


語言本身的學習
C語言：

除了最最基礎的語法：變量、條件、循環、字符串、數組、函數、結構體等之外

C語言最最最最最重要的那就是：指針、內存管理，以後企業里開發就靠它倆吃飯，這也是檢驗C語言掌握情況的兩大標準

推薦書籍：《C Primer Plus》、《C和指針》、《C專家編程》

C++：

C++和C語言確實是不同的語言，但是C++確實是對C語言的延伸，可以理解為在C語言裡加入了面向對象的特性。因為只有面向對象特性的加持，代碼才能更好的復用、擴展和工程化，這是大型項目的必備要素

除了C語言所有的底子之外，還需要學習C++的面向對象（封裝、繼承與多態）特性、泛型、模板、STL等等

推薦書籍（有先後順序）：《C++ Primer》、《Effective C++》、《C++ 標準程序庫》、《STL源碼剖析》《深度探索C++對象模型》

最後一個小建議是：語言部分的學習建議不要拖太久，一定要規劃好時間，一鼓作氣，高強度給它壓下來，否則容易把自己搞泄氣。


編程基礎四大件
基礎四大件包括：數據結構和算法、計算機網絡、作業系統、設計模式

這跟學什麼程式語言、後續從事什麼編程方向均無關，只要做編程開發，這四個計算機基礎就無法避開。可以這麼說，這基礎四大件真的比程式語言重要！！！

1、數據結構和算法

可以說這個直接決定了面試的成敗！幾種基礎數據結構類型得爛熟於心，比如：字符串、鍊表、二叉樹、堆、棧、隊列、哈希等；基本的幾大算法也要了如指掌，比如查找、排序、動態規劃、分治等等。

建議LeetCode多刷題。

參考資料：《大話數據結構》、《算法》、《劍指offer》《LeetCode刷題》

2、計算機網絡

此處的計算機網絡指的就是TCP/IP協議棧，可以說它是當下網際網路通信的基石，無論如何一定要對TCP/IP的協議棧了如指掌，主要就是學習和掌握原理，包括：ARP協議、IP協議、ICMP協議、TCP和UDP協議、DNS協議、HTTP協議、HTTPS協議。

推薦書籍：《TCP/IP詳解》

3、作業系統

該部分重點包括：進程和線程的相關原理（原子性、並發、鎖）、內存相關原理（內存分布、內存調度）

推薦書籍：《深入理解計算機系統》

4、設計模式

倒不需要23種設計模式全部記住，常見的幾個如：單例模式、工廠模式、代理模式、策略模式、模板方法模式建議熟練於心。

推薦書籍：《大話設計模式》、《設計模式之禪》


應用實踐編程
這部分會涉及到一些工具、編程環境、和具體編程實踐，應該說這一部分學完，自己應該能做點東西出來、或者說能看懂一些實際項目的代碼。

實踐這部分，我也是結合我之前在通信公司的實際工作經歷和所聽所見，來聊一聊。

1、Linux作業系統的使用

很多人初學C語言、C++（包括我）都是在Windows環境上進行的，而實際企業級開發幾乎不可能，所以Linux系統必須要會，我們別無選擇。先談使用。

包括：常見的Linux操作命令、基本的Shell編程。

推薦書籍：《鳥哥的Linux私房菜》

2、編譯/調試工具

首先是跟編譯相關的：編譯工具！

我們知道很多人學C語言、C++都在類似Visual Studio這種集成IDE里進行代碼編譯，這個其實也用了編譯器，只不過是微軟自家的MS編譯器，而且所有操作均可視化。而企業里開發很少會基於Windows系統，所以Linux平台上的編譯器更為重要，最典型的當屬GCC，甚至有些公司有自己定製過的交叉編譯工具，但沒關係，只要GCC熟悉，其他問題都不大。

其次，大家自學C語言、C++，都藉助類似VS這種IDE，點按鈕即可對源文件編譯。而企業里實際項目的編譯動作叫make，編譯的實際動作和過程都是寫在makefile文件里，所以makefile的書寫規則建議學習！

最後說到調試，Linux平台上的GDB調試工具要熟練使用，會藉助於它進行調試。

推薦資料：這部分沒有書可推薦，英文好的同學可以直接看GNU官網關於GCC和GDB的文檔，中文材料可以看：《debuging with gdb》（中文版）和陳皓先生的《跟我一起寫makefile》

3、Linux環境編程

還是那句話，企業里C語言和C++幾乎都是基於Linux平台的，這一部分我們沒辦法避開

（1）Linux系統編程

眾所周知，Windows系統編程里有各種各樣的Windows API，同理Linux系統API的使用就更加重要。

（2）多線程編程

此處指的是多線程編程實踐相關的東西，一般包括：線程、資源、信號、同步、互斥、鎖等等一些具體的編程方法。

（3）網絡編程

此處的網絡編程主要指的是具體Linux系統上的網絡編程API和IO函數的編程實踐。

推薦書籍：這三部分綜合在一起，推薦必看書籍包括《Unix環境高級編程》、《Linux高性能伺服器編程》、《POSIX多線程程序設計》
